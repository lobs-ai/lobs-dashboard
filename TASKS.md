# MoltBot Dashboard — Usage & Feature Summary

## Purpose

The MoltBot dashboard is the **single control center** for interacting with MoltBot. It replaces scattered tools (Discord threads, repos, notes) with one place to **see state, assign work, review outputs, and manage projects**—without MoltBot needing dangerous system access.

---

## How You’ll Use the Dashboard (Day-to-Day)

1. **Open the dashboard**
   - Immediately see what MoltBot is doing now, what’s queued, and what’s blocked.
   - No chat-first workflow—status and artifacts come first.

2. **Assign tasks**
   - Create tasks directly in the dashboard (research, coding, planning, analysis).
   - Tasks become structured objects, not free-form chat messages.

3. **Review outputs**
   - MoltBot produces files, notes, diffs, or summaries.
   - You review and approve/reject before anything is merged or finalized.

4. **Track progress over time**
   - See completed work, abandoned attempts, and rationale.
   - MoltBot can reference prior work without you re-explaining context.

5. **Lightweight interaction**
   - Use Discord only for notifications and quick questions.
   - The dashboard is the “source of truth.”

---

## Core Dashboard Features

### 1. Task System

- Structured tasks with:
  - Title
  - Type (research, coding, planning, review, etc.)
  - Status (queued, active, blocked, done)
  - Priority
- MoltBot can:
  - Propose new tasks
  - Reorder tasks
  - Split tasks into subtasks
- You retain **final approval**.

---

### 2. Work State & Transparency

- Shows:
  - Current task MoltBot is working on
  - Why it chose that task
  - What assumptions it’s using
- Prevents “black box” behavior.

---

### 3. File & Artifact Viewer

- Browse MoltBot-generated artifacts:
  - Research notes
  - Markdown docs
  - Code diffs
  - Data summaries
- Files live in:
  - A shared GitHub repo (for you)
  - Internal private storage (for MoltBot state)
- Dashboard knows **how to render different file types**.

---

### 4. GitHub Integration (Controlled)

- Read-only access to most repos
- Write access only where explicitly allowed
- MoltBot can:
  - Propose commits
  - Open pull requests
  - Explain changes before submission
- You approve before merge.

---

### 5. Research Management

- Research is treated as a first-class object:
  - Links
  - Short metadata (title, difficulty, relevance)
  - Notes generated by MoltBot
- Examples:
  - SWE-bench experiments
  - Paper summaries
  - LeetCode problem tracking
- Minimal structure, maximum flexibility.

---

### 6. Block-Based UI Elements

- Certain items appear as draggable blocks:
  - Tasks
  - Research items
  - “To review” artifacts
- You can:
  - Mark complete
  - Defer
  - Archive
- Supports visual organization without complexity.

---

### 7. History & Memory

- Full timeline of:
  - What MoltBot did
  - What worked
  - What failed
- Prevents repeated mistakes.
- Enables higher-level reasoning and long-term planning.

---

### 8. Security & Trust Model

- MoltBot:
  - Runs as a restricted user
  - No sudo
  - No direct access to secrets
- Dashboard is your **approval gate**.
- MCP servers and tools run under separate users.

---

## What the Dashboard Is _Not_

- Not a chat app replacement
- Not a cluttered “everything everywhere” UI
- Not autonomous without oversight

---

## Design Philosophy

- **One app**
- **Clear state**
- **Human-in-the-loop**
- **Artifacts over chatter**
- **Power without risk**

---

## End Goal

The dashboard makes MoltBot feel like a **real assistant**:

- Proactive but constrained
- Capable but transparent
- Helpful without being dangerous

You don’t manage MoltBot.
You _collaborate_ with it.
